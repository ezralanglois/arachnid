align

Align a set of particle windows to a reference and reconstruct the windows into a new volume

This pySPIDER batch file (spi-align) performs reference-based alignment on a stack of projections, then reconstructs them into a volume ready for refinement.

Tips

  1. If the data stack has already been phase flipped, then set –phase-flip to True
  2. If the data stack is not ctf corrected, then you must specify a defocus-file. If the stacks are organized by defocus group or micrograph, then this is all that is required. However, if you are using a single stack then you must specify stack-select to map each particle back to its micrograph.
  3. Be aware that if the data or the reference does not match the size in the params file, it will be automatically reduced in size Use –bin-factor to control the decimation of the params file, and thus the reference volumes, masks and data stacks.
  4. When using MPI, home-prefix should point to a directory on the head node, local-scratch should point to a local directory on the cluster node, shared-scratch should point to a directory accessible to all nodes and all other paths should be relative to home-prefix.

Examples

# Source AutoPart - FrankLab only

$ source /guam.raid.cluster.software/arachnid/arachnid.rc

# Run alignment over a phase flipped stack

$ spi-align image_stack_*.ter -p params.ter -o alignment_0001.ter -r ref_vol_0001.spi --phase-flip

Critical Options

-i <FILENAME1,FILENAME2>, --input-files <FILENAME1,FILENAME2>, FILENAME1 FILENAME2

List of input filenames containing stacks of projection images If you use the parameters -i or –inputfiles they must be comma separated (no spaces). If you do not use a flag, then separate by spaces. For a very large number of files (>5000) use -i “filename*”

-o <FILENAME>, --output <FILENAME>

Output filename for the raw full and half volumes as well as base output name for FSC curve (res_$output)

-p <FILENAME>, --param-file <FILENAME>

Path to SPIDER params file

--bin-factor <FLOAT>

Number of times to decimate params file

--phase-flip <BOOL>

Set to True if your data stack(s) has already been phase flipped or CTF corrected (Default: False)

-r <FILENAME>, --reference <FILENAME>

Filename for reference with the proper pixel size

Useful Options

--max-ref-proj <INT>

Maximum number of reference projections in memory

--use-apsh <BOOL>

Set True to use AP SH instead of AP REF (trade speed for improved accuracy) (Default: False)

--use-flip <BOOL>

Use the phase flipped stack for alignment (Default: False)

--prep-thread <INT>

Number of threads to use for volume preparation (Default: 0, use all threads)

Volume Projection Options

These options are passed to SPIDER’s pj_3q command, the default values are generally fine for most experiments.

--pj-radius <float>

Radius of sphere to compute projection, if less than one use 0.69 times the diameter of the object in pixels (Default: -1)

Angular Sampling Options

These options are passed to SPIDER’s vo_ea command, the default values are generally fine for most experiments. The most common parameter to change is theta-delta, which controls the sampling rate of the Euler sphere.

--theta-delta <float>

Angular step for the theta angles (Default: 15.0)

--theta-start <float>

Start of theta angle range (Default: 0.0)

--theta-end <float>

End of theta angle range (Default: 89.9)

--phi-start <float>

Start of phi angle range (Default: 0.0)

--phi-end <float>

End of phi angle range (Default: 359.9)

Alignment Options

These options are passed to SPIDER’s ap_sh or ‘ap_ref’ command, the default values are generally fine for most experiments.

--angle-range <FLOAT>

Maximum allowed deviation of the Euler angles, where 0.0 means no restriction (Default: 0.0)

--angle-threshold <FLOAT>

Record differences that exceed this threshold (Default: 1.0)

--trans-range <INT>

Maximum allowed translation; if this value exceeds the window size, then it will lowered to the maximum possible (Default: 24)

--trans-step <INT>

Translation step size (Default: 1)

--first-ring <INT>

First polar ring to analyze (Default: 1)

--ring-last <INT>

Last polar ring to analyze; if this value is zero, then it is chosen to be the radius of the particle in pixels (Default: 0)

--ring-step <INT>

Polar ring step size (Default: 1)

--ray-step <INT>

Step for the radial array (Default: 1)

--test-mirror <BOOL>

If true, test the mirror position of the projection (Default: True)

Other Options

This is not a complete list of options available to this script, for additional options see:
