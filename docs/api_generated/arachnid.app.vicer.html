vicer

Clean particle selection with the view classifier or ViCer

This script (ara-vicer) was designed to post clean an existing particle selection. It requires that the projections be grouped by view as well as 2D alignment parameters. This can be obtained by 2D-reference free classification or 3D orientation determination.

Notes

  1. Filenames: Must follow the SPIDER format with a number before the extension, e.g. mic_00001.spi. Output files just require the number of digits: –output sndc_0000.spi
  2. Parallel Processing - Several micrographs can be run in parallel (assuming you have the memory and cores available). -p 8 will run 8 micrographs in parallel.
  3. Supports both SPIDER and RELION alignment files - if the alignment comes from pySPIDER, you must specified –scale-spi or set True.

Examples

# Run with a disk as a template on a raw film micrograph

$ ara-vicer -i image_file_0001.spi -a align_file.spi -o output/clean_000.spi -p params.spi -w 10

Critical Options

-i <FILENAME1,FILENAME2>, --input-files <FILENAME1,FILENAME2>, FILENAME1 FILENAME2

List of input stack filenames or single input stack filename template If you use the parameters -i or –inputfiles the filenames may be comma or space separated on the command line; they must be comma seperated in a configuration file. Note, these flags are optional for input files; the filenames must be separated by spaces. For a very large number of files (>5000) use -i “filename*”

-o <FILENAME>, --output <FILENAME>

Output filename for the output embeddings and selection files (prefixed with sel_$output)

-p <FILENAME>, --param-file <FILENAME>

Filename for SPIDER parameter file describing a Cryo-EM experiment

-a <FILENAME>, --alignment <FILENAME>

Input file containing alignment parameters

Useful Options

These options

--order <INT>

Reorganize views based on their healpix order (overrides the resolution parameter)

--prob-reject <FLOAT>

Probablity that a rejected particle is bad

--neig <INT>

Number of eigen vectors to use

--expected <FLOAT>

Expected fraction of good data

--resolution <FLOAT>

Filter to given resolution - requires apix to be set

Customization Options

Generally, these options do not need to be changed, unless you are giving the program a non-standard input.

--disable-rtsq <BOOL>

Do not use alignment parameters to rotate projections in 2D

--scale-spi <BOOL>

Scale the SPIDER translation (if refinement was done by pySPIDER)

Testing Options

Generally, these options do not need to be considered unless you are developing or testing the code.

--single-view <INT>

Test the algorithm on a specific view

--random-view <INT>

Set number of views to assign randomly, 0 means skip this

--disable-bispec <BOOL>

Disable bispectrum feature space

--nsamples <INT>

Number of rotational samples

--angle-range <FLOAT>

Angular search range

Other Options

This is not a complete list of options available to this script, for additional options see:

  1. Options shared by all scripts ...
  2. Options shared by MPI-enabled scripts...
  3. Options shared by file processor scripts...
  4. Options shared by SPIDER params scripts...
